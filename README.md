dicas para estudo de teste e qualidade

- Conhecimento em lógica de Programação;
    * Ruby 
    * Java 
    * Javascript 
- Boa análise de histórias de usuário;
    * Conhecimento em BDD 
    * Conhecimento prévio de Cucumber 
Conhecimento básico dos ambientes e ferramentas de desenvolvimento dos aplicativos para iOS e Android e/ou de desenvolvimento web;
    * Git 
    * Xcode 
    * Android Studio 
    * Appium com qualquer linguagem de programação; 
    * Selenium Webdriver com qualquer linguagem de programação 
    * Protractor 
- Conhecimento em Testes de API:
    * Httparty 
    * Pacote nodejs (mocha, chai e supertest) 
    * RestAssured 
- Conhecimento em Integração e Entrega contínua;
    * Jenkins 
    * Traves 
    * Circle CI



Nível 0 
- [ ]  Básico em lógica de programação
- [ ]  Sabe realizar testes exploratórios e manuais
- [ ] Sabe reportar Bug simples (Front)
- [ ] Tem vontade de aprender
- [ ] Sabe buscar ajuda dos mais seniors
- [ ] Escreve cenários de testes funcionais (junto PO e Designer)
- [ ] Sabe gerenciar as principais massa de dados para os testes
- [ ] Conhecimentos básicos de agilidade


Nível 1 

- [ ] Sabe reportar qualquer tipo de Bug (Backend, ambiente, Front)
- [ ] Sabe usar melhores evidencias para reportar bugs (logs)
- [ ] Desenvolve em alguma linguagem de programação (Java, Ruby, js, etc..)
- [ ] Escreve cenários para diferentes tipos de testes (junto todo o time, aplicar BDD)
- [ ] Sabe gerenciar as principais massa de dados para os testes
- [ ] Conhecimentos básicos de agilidade e produto
- [ ] Possui uma boa comunicação com o time


Nível 2
 

- [ ] Sabe usar melhores evidências ao reportar bugs (log por exemplo)
- [ ] Escreve cenários para diferentes tipos de testes
- [ ] Compartilha cenários com todo o time (aplicar BDD)
- [ ] Automatiza testes E2E, UI
- [ ] POO
- [ ] Versionamento com Git
- [ ] Faz pair programing com os desenvolvedores
- [ ] Sabe usar os frameworks de desenvolvimento (Xcode e Android Studio)
- [ ] Usa ferramentas para testes de API (Postman, Swagger, etc..)
- [ ] Realiza a estratégia de testes para time (levantamento de massa de dados, ambiente, gerencia versões, etc..)
- [ ] Sabe gerenciar  ambientes para os testes
- [ ] Tem conhecimento em agilidade e produto (senso de dono)
- [ ] Possui uma boa comunicação com Stakeholders


Nível 3

- [ ]  Sabe realizar debug de bugs complexos
- [ ] Sabe usar melhores evidencias em reporte de bugs, inclusive produção (log por exemplo, ferramentas de monitoria)
- [ ] É referência técnica para outros QAs
- [ ] Escreve cenários para diferentes tipos de testes (é referencia para o time)
- [ ] Faz pair programing com os desenvolvedores
- [ ] Sabe realizar e aplicar testes não funcionais (performance, estresse, segurança, acessibilidade )
- [ ] Compartilha cenários com todo o time (é referência da técnica BDD)
- [ ] Automatiza qualquer tipo de teste necessário para o contexto do time (unitário, API, contrato, não funcional, etc..)
- [ ] Tem condições de aplicar um CI, ajudar CD, pair com Devops
- [ ] Sabe usar os frameworks de desenvolvimento (Xcode e Android Studio)
- [ ] Usa ferramentas para testes  de API (Postman, Swagger, etc..)
- [ ] Automação a nível de API e Contrato
- [ ] Sabe gerenciar as principais massa de dados, ambientes para os testes e integração com outros serviços
- [ ] É referência de agilidade no time
- [ ] Ajudar mediar conflitos com Stakeholders
- [ ] Compartilha conhecimento para outros integrantes do capitulo
- [ ] Ter conhecimento em testes de outras plataformas WEB/Mobile
